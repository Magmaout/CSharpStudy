Задание: Разработка ядра системы "GeoCalc Pro"

Вы работаете стажером в инженерном бюро. Вам поручили написать прототип модуля для расчета параметров строительных участков. Участки могут иметь разную форму (круг, прямоугольник, треугольник). Система должна уметь хранить фиксированный набор участков, сравнивать их площадь, считать периметр (для забора) и площадь (для покраски), а также выполнять арифметические операции над их характеристиками.

Технические требования:
1. Иерархия классов (Наследование и Полиморфизм)
Создайте абстрактный базовый класс Figure.
Свойства: Color (строка), Name (строка).
Абстрактные методы: GetArea() (возвращает double), GetPerimeter() (возвращает double).
Виртуальный метод: GetInfo(), который возвращает строку с описанием фигуры.
Создайте три класса-наследника:
Circle (свойство Radius).
Rectangle (свойства Width, Height).
Triangle (свойства A, B, C — длины сторон). Подсказка: используйте формулу Герона для площади.
2. Интерфейсы
Создайте интерфейс ICostable.
Метод: CalculateMaterialCost(double pricePerUnit).
Логика: Все фигуры должны реализовывать этот интерфейс. Метод должен возвращать стоимость покрытия площади фигуры (Площадь * Цена).
3. Перегрузка операторов
В абстрактном классе Figure (или в каждом наследнике, если логика различается, но лучше в базовом для полиморфизма) перегрузите следующие операторы:
> и <: Сравнивают две фигуры по их Площади. Возвращают bool.
+: Складывает Периметры двух фигур. Возвращает double (общий периметр).
==: Проверяет, равны ли площади двух фигур (с точностью до 0.01).
4. Управление данными (Массивы)
Создайте класс FigureStorage (Хранилище).
Внутри него должно быть приватное поле: private Figure[] _figures;.
В конструкторе инициализируйте массив фиксированным размером (например, 10 элементов).
Метод AddFigure(Figure f): Добавляет фигуру в первую свободную ячейку массива. Если массив полон — выводит сообщение об ошибке в консоль.
Метод GetAll(): Возвращает массив всех добавленных фигур (можно вернуть копию или ссылку, но без null элементов, либо с обработкой null в цикле).
Метод GetTotalArea(): С помощью цикла for считает суммарную площадь всех фигур в массиве.
5. Консольное приложение (Main)
В методе Main реализуйте простое меню:
Добавить фигуру (пользователь выбирает тип и вводит параметры).
Показать все фигуры (вызывает полиморфный метод GetInfo()).
Сравнить две фигуры (пользователь вводит индексы фигур в массиве, программа пишет, какая больше).
Посчитать стоимость материалов (вводит цену за кв.м., программа считает для каждой фигуры через интерфейс).
Выход.
